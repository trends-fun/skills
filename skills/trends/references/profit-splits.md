# 分成解释：用用户能听懂的话说

这里专门处理用户问“我怎么多赚一点”“分成怎么填”“dev-bps 是啥”“project-submitter-bps 是啥”。

默认先引导用户使用 IAO / 项目币。普通发币是 Legacy 路径，只在用户明确要普通 meme 币、推特相关代币、不绑定项目、不要 IAO 时使用。

## 1. Legacy 普通发币的分成

Legacy 普通发币时，用户真正关心的是：

- 我作为发币人能拿多少。
- X profile / 推文对应的一方能拿多少。

不要直接问：

- `dev-bps 填多少？`

应该问：

- `你希望自己拿多少分成？默认你拿 96%，X/推特作者拿 4%。`

内部换算：

| 用户说法 | 命令参数 |
| --- | --- |
| 我拿 100% | 没有 X 链接时不用传；有 X 链接时用 `--dev-bps 10000` |
| 我拿 96%，对方 4% | `--dev-bps 9600` |
| 我拿 90%，对方 10% | `--dev-bps 9000` |
| 我拿 80%，对方 20% | `--dev-bps 8000` |

规则：

- 填了 X profile 或推文链接时，分成才会拆成两边。
- 不填 X 链接时，`--dev-bps` 会被忽略，实际就是发币人 100%，外部对象 0%。
- `--dev-bps` 是 0 到 10000 的整数；10000 代表 100%。
- 默认 `--dev-bps 9600` 代表发币人 96%，X/推特作者 4%。

向用户展示时写：

- `你拿 96%，X/推特作者拿 4%（内部参数 --dev-bps 9600）。`

## 2. 默认路径：项目币 / IAO 的分成

项目币场景里，用户真正关心的是：

- 项目方拿多少。
- 我作为发币/agent 的人拿多少。

不要直接问：

- `project-submitter-bps 填多少？`

应该问：

- `项目方拿多少分成？默认项目方拿 70%，你拿 30%。`

内部换算：

| 用户说法 | 命令参数 |
| --- | --- |
| 项目方 70%，我 30% | `--project-submitter-bps 7000` |
| 项目方 68%，我 32% | `--project-submitter-bps 6800` |
| 项目方 50%，我 50% | `--project-submitter-bps 5000` |
| 项目方 100%，我 0% | `--project-submitter-bps 10000` |

规则：

- `--project-submitter-bps` 表示项目方拿多少。
- 用户自己拿 `10000 - project-submitter-bps`。
- 默认 `7000`，即项目方 70%，用户 30%。
- 最低 `5000`，即项目方至少 50%。
- 最高 `10000`，即项目方 100%，用户 0%。
- 如果用户不回答分成问题，用默认 70% / 30%，但预检里必须写清楚。

向用户展示时写：

- `项目方拿 70%，你拿 30%。`

## 3. 预检必须说清楚

Legacy 普通发币预检必须包括：

- 币名
- 符号
- 简介
- 图标来源
- X 链接或明确不填
- 首买金额
- 你自己的分成比例
- 对方分成比例

项目币预检必须包括：

- 当前执行钱包地址或 keypair 上下文
- 项目链接
- 币名
- 符号
- 简介
- 图标来源：项目封面或本地图片
- 首买金额
- 项目方分成
- 用户分成

确认前不要进入真实写操作。
